# LightRAW 阶段二规格与验收

## 交付边界

阶段二在阶段一解码管线之上新增以下非破坏性编辑指令：

1. 完整基本面板：白平衡、曝光、对比度、高光、阴影、白色色阶、黑色色阶、纹理、清晰度、去雾、自然饱和度和饱和度。
2. 色调曲线：RGB 主曲线及红、绿、蓝分通道曲线，每条曲线最多 16 个控制点。
3. HSL：红、橙、黄、绿、青、蓝、紫、洋红八个颜色族的色相、饱和度和明亮度。
4. 实时直方图：对当前裁剪区域抽样，并应用当前调色配方后重新计算 RGB 分布。
5. 细节：锐化数量、半径、细节，以及明亮度和颜色降噪。
6. 裁剪与几何：自由裁剪、1:1/4:3/16:9/3:2、左右 90° 旋转、水平/垂直翻转和 ±45° 拉直。

蒙版、图层、历史栈、预设、图库、批量处理和最终导出仍属于后续阶段。

## 实现约束

- `DevelopRecipe` 是版本为 2 的可 JSON 序列化编辑配方；UI 不直接持有 shader 状态。
- WebGL 2 使用原始 RGBA16F 图像纹理、RGBA16F 曲线 LUT 和邻域采样进行实时预览。
- 曲线采用分段线性插值，避免控制点之间过冲；HSL 使用重叠的三角形色相权重，避免颜色族边界产生硬断层。
- 直方图最多抽样 2.4 万个像素，并在滑块变化后延迟 80 ms 计算，避免阻塞连续拖动。
- 裁剪坐标始终保存为原图归一化坐标；UI 在旋转和翻转时执行显示坐标与源坐标的双向映射。

## 本地运行

安装依赖后运行开发版：

```bash
npm ci
npm run tauri dev
```

生成 macOS 开发版应用：

```bash
npm run tauri -- build --debug --bundles app
open src-tauri/target/debug/bundle/macos/LightRAW.app
```

原生 LibRaw/libheif 依赖及 Windows 环境准备与阶段一相同，见仓库根目录 `README.md`。

npm run typecheck

## 已验证

- 36 个前端单元测试通过，覆盖配方、参数范围、曲线、HSL、裁剪与拉直采样、调色与细节参考实现和实时直方图。
- Rust 解码测试通过，`clippy -D warnings` 无警告。
- macOS 原生 `.app` 成功加载系统 2880×2880 HEIC 样片，GLSL 在 WKWebView 中编译并显示。
- 曝光调整、编辑后直方图、RGB 曲线控制点及 HSL 单色通道均在真实应用中生效。
- 16:9 预设、90° 旋转和旋转后的自由裁剪手柄均完成可视验收。
- 当前 Mac 上单次编辑到下一次合成机会约 18–25 ms；这是应用内近似指标，不是显示器端到端仪器测量。

## 已知限制

- Windows 实机仍未验证；跨平台源码和构建配置保持不变。
- 纹理、清晰度、锐化和降噪是面向实时预览的轻量邻域算法，不等同于 Lightroom 的私有算法。
- 拉直会自动放大以避免空白边角；暂不提供自动水平线检测。
- 为保证裁剪框与最终采样严格一致，拉直角度非零时显示最终裁剪预览并暂时隐藏自由手柄；角度归零后手柄恢复。
- 阶段二仍只在内存中保存编辑配方，持久化和历史记录在阶段四实现。
