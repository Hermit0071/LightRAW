# RapidRAW 参考改版与本地能力计划

## 实施边界

LightRAW 采用独立实现：只参考 RapidRAW 的公开界面和用户可观察行为，不复制其 AGPL 源码、资源或品牌元素。目标是获得相近的摄影工作流，同时继续使用 LightRAW 自有的光圈标识、配色、数据模型和渲染管线。

云同步、在线图库、在线推理和账户服务不纳入范围。需要模型的能力在首次使用时下载模型包到应用数据目录；下载完成后可断网运行，也可单独删除模型包。

## 体积预算

- 当前 macOS 调试应用：约 50 MB。
- 核心安装包目标：不超过 150 MB，不内置 AI 权重。
- RAW、HEIF、GPU 调色、图库、导出等日常能力属于核心包。
- 智能蒙版、语义搜索、修复、深度和 AI 降噪作为独立模型包；模型体积不计入核心安装包，并在下载前显示实际大小。

## 交付顺序

1. 工作台与文件管理：完成真实文件夹导航、相册/标签、虚拟副本和 EXIF 浏览。
2. 调色与效果：在现有解耦色彩引擎中补齐 GPU 效果、示波器、镜头与几何能力。
3. 本地组合工作流：HDR、全景、拼贴与底片转换。
4. 模型管理器与本地 AI：先实现安装、校验、版本和卸载，再逐项接入蒙版、搜索、修复和降噪。
5. 发布工程：Windows 实机回归、原生依赖随包分发、签名、公证和安装包体积门禁。

每个步骤都以可运行版本交付；未实现的能力不会用无效按钮或假数据占位。
